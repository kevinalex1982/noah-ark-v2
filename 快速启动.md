## 🚀 OpenClaw 网关

### 状态检查

```powershell
# 检查网关状态
openclaw gateway status

# 检查完整状态（含安全审计）
openclaw status
```

### 启动/停止/重启

```powershell
# 启动后台服务
openclaw gateway start

# 重启后台服务
openclaw gateway restart

# 停止后台服务
openclaw gateway stop
```

### 前台运行（调试用）

```powershell
# 前台启动网关（会阻塞终端）
openclaw gateway

# 查看详细日志
openclaw logs --follow
```

### 服务信息

- **服务类型**: Windows Scheduled Task
- **服务文件**: `~\.openclaw\gateway.cmd`
- **日志文件**: `C:\Users\ADMINI~1\AppData\Local\Temp\openclaw\openclaw-YYYY-MM-DD.log`
- **监听地址**: `0.0.0.0:18789`
- **仪表盘**: http://192.168.3.201:18789/

## 🚀 诺亚方舟 Next.js 应用

### ⚠️ 重要提示

**永远不要用 `exec` 启动长期服务！** 这会导致 OpenClaw 崩溃。

**正确方式**: 用户手动启动，Agent 只负责测试。

### 启动方式

```powershell
# 方式一：使用启动脚本（推荐）
cd C:\Users\Administrator\.openclaw\workspace\noah-ark-v2
.\start-dev.ps1

# 方式二：直接运行 Next.js
.\node_modules\.bin\next dev -p 3001

# 方式三：使用 npm（如果 node_modules/.bin 不可用）
npm run dev
```

### 服务信息

- **框架**: Next.js 15
- **端口**: 3001
- **工作目录**: `C:\Users\Administrator\.openclaw\workspace\noah-ark-v2`

### 测试服务（不阻塞）

```powershell
# 使用 Invoke-WebRequest 测试（推荐）
Invoke-WebRequest -Uri "http://localhost:3001" -UseBasicParsing

# 测试 API 端点
Invoke-WebRequest -Uri "http://localhost:3001/api/health" -UseBasicParsing
```

## 🐛 故障排查

### OpenClaw 崩溃/无法启动

```powershell
# 1. 检查是否有残留进程
tasklist | findstr node

# 2. 查看最新日志
Get-Content "C:\Users\ADMINI~1\AppData\Local\Temp\openclaw\openclaw-$(Get-Date -Format 'yyyy-MM-dd').log" -Tail 100

# 3. 尝试重启服务
openclaw gateway restart

# 4. 如果还不行，完全重启
openclaw gateway stop
Start-Sleep -Seconds 2
openclaw gateway start
```

### 诺亚方舟服务无法启动

```powershell
# 1. 检查端口是否被占用
netstat -ano | findstr :3001

# 2. 杀死占用端口的进程（替换 PID）
taskkill /F /PID <PID>

# 3. 清理 .next 缓存
Remove-Item -Recurse -Force .next

# 4. 重新启动
.\start-dev.ps1
```

### 权限错误 (missing scope)

如果看到 `missing scope: operator.read` 错误：
1. 检查 `~/.openclaw/openclaw.json` 中的 scopes 配置
2. 确保控制 UI 使用的凭证有正确的权限范围
