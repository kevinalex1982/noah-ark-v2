# 虹膜设备返回错误分析

## 错误码对照表

| errorCode | 含义 | 原因 | 解决方案 |
|-----------|------|------|----------|
| 0 | 成功 | - | - |
| 5 | Member save state error | 设备未锁定就执行添加操作 | 先调用 memberSaveState 锁定设备 |
| 6 | Left iris cannot be empty | 左眼虹膜图片为空 | 检查 irisLeftImage 是否有数据 |
| 16 | Member not found | 人员不存在 | 先添加人员 |
| 97 | App not be inited | 设备忙 | 等待后重试 |
| 98 | App game over | 设备状态异常 | 重置设备 |

## 超时错误分析

| 超时类型 | 可能原因 | 排查方向 |
|----------|----------|----------|
| 请求超时 (10s+) | 图片数据量大，处理慢 | 检查图片大小、BMP转换 |
| 连接超时 | 网络问题、设备离线 | ping设备IP |
| 锁定超时 | 设备正在处理其他请求 | 等待设备空闲 |

## 常见错误场景

### 1. errorCode=5 (Member save state error)

**现象**：设备返回 errorCode=5

**原因**：调用 memberSave 接口前没有锁定设备

**正确流程**：
```
1. 调用 memberSaveState(state=1) 锁定设备
2. 等待 500ms
3. 调用 memberSave 上传数据
4. 等待 500ms
5. 调用 memberSaveState(state=0) 解锁设备
```

### 2. 请求超时

**现象**：请求超过15秒无响应

**可能原因**：
1. 人脸图片处理失败（图片格式不对、大小过大）
2. 虹膜图片BMP转换失败
3. 网络问题

**排查步骤**：
1. 检查 `data/debug/` 目录下的请求日志
2. 检查控制台 `[FaceImage]` 日志，确认人脸图片读取成功
3. 检查控制台 `[ImageConvert]` 日志，确认BMP转换成功

### 3. errorCode=6 (Left iris cannot be empty)

**现象**：设备返回左眼虹膜为空

**原因**：
1. MQTT消息中 content 字段为空
2. content 解析失败

**排查步骤**：
1. 检查 MQTT 消息的 content 字段
2. 检查 `data/mqtt_logs/` 目录下的日志文件

## 日志文件位置

| 日志类型 | 位置 | 内容 |
|----------|------|------|
| MQTT消息 | `data/mqtt_logs/` | MQTT收到的原始消息 |
| 调试日志 | `data/debug/` | 发送给虹膜设备的请求JSON |
| 测试报告 | `docs/测试/` | 测试执行记录 |

## 注意事项

1. **人脸图片必须从文件读取**，不能硬编码（血泪教训）
   - 文件路径：`data/face_photo_sample.txt`
   - 代码使用 `getSampleFaceImage()` 函数读取
   - 超时错误很可能是人脸图片读取失败

2. **锁定和添加之间必须等待500ms**
3. **添加和解锁之间必须等待500ms**
4. **删除操作不需要锁定**

## 测试前准备

⚠️ **重要：测试开始前必须清空设备上的测试数据！**

虹膜设备不允许重复添加相同 person_id 的数据：
- 如果设备上已存在 person_id=123456，再次添加会失败
- 测试脚本开始前需要先删除设备上的测试数据

清空方法：
1. 使用 `clearIrisDevice()` 清空设备
2. 或手动调用 `memberDelete` 接口删除 person_id=123456
