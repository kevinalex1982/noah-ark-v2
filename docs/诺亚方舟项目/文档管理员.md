## 🎯 文档使用原则

### 1. 每个文档只讲自己的职责

| 内容 | 查看文档 |
|------|----------|
| 数据库表结构 | 数据库设计文档 1.0.md |
| 业务逻辑 | Next.js 后端设计文档.md |
| 测试数据 | IAMS 模拟服务设计文档 1.0.md |
| API 接口 | 后端 API 接口文档.md |
| faceImage | Next.js 后端设计文档.md |
| 启动方式 | PROJECT_RULES.md 或 快速启动.md |

### 2. 避免重复内容
- ❌ 不要在其他文档重复定义数据库表结构
- ❌ 不要在其他文档重复测试数据
- ❌ 不要在其他文档重复 faceImage 说明

### 3. 修改文档前先确认
- 修改表结构 → 只改数据库设计文档
- 修改 API → 只改后端 API 接口文档
- 修改业务逻辑 → 只改 Next.js 后端设计文档
