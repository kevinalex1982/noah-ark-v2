## ⚠️ 血泪教训（2026-03-25，永远不要忘记！）

### 后端崩溃会导致 OpenClaw 一起挂掉

**原因**：诺亚后端和 OpenClaw 共用同一个 Node.js 进程，后端未捕获的异常会导致整个进程退出。

**解决方案**：
1. **后端必须加全局错误边界** - API 路由、数据库操作、定时任务都要 try-catch
2. **Next.js 配置错误处理** - 确保 `next.config.mjs` 和 `app/layout.tsx` 有错误边界
3. **数据库迁移要兼容** - 旧数据库文件可能缺少新列，需要迁移脚本或重建

**如果 OpenClaw 崩了**：
```powershell
# 1. 查看诺亚后端日志
Get-Content " noah-ark-v2\next.log" -Tail 50

# 2. 查看 OpenClaw 日志
Get-Content "C:\Users\ADMINI~1\AppData\Local\Temp\openclaw\openclaw-$(Get-Date -Format 'yyyy-MM-dd').log" -Tail 100

# 3. 重启 OpenClaw
openclaw gateway restart

# 4. 如果是数据库错误，删除旧数据库重建
# （先备份！）
$oldDb = " noah-ark-v2\data\noah-ark.db"
Copy-Item $oldDb "$oldDb.backup.$(Get-Date -Format 'yyyyMMdd-HHmmss')" -Force
Remove-Item $oldDb -Force
```

## 🚀 完整启动流程（测试 IAMS 功能）

### 步骤 1：启动 MQTT Broker（模拟 IAMS）

**为什么需要**：MQTT broker 模拟 IAMS 上级平台，用于接收和下发凭证指令。

**启动命令**：
```powershell
cd  noah-ark-v2
.\start-mqtt.ps1
```

**预期输出**：
```
========================================
  MQTT Broker (模拟 IAMS)
========================================

端口：1883
协议：TCP/MQTT

按 Ctrl+C 停止服务

[MQTT Broker] ✅ 服务器启动成功，端口：1883
[MQTT Broker] 🎯 等待客户端连接...
```

**特点**：
- ✅ **前台运行** - 可以看到实时日志
- ✅ **显示连接** - 看到客户端连接/断开
- ✅ **显示消息** - 看到收发的 MQTT 消息
- ❌ **占用窗口** - 需要保持窗口打开

**停止**：按 `Ctrl+C` 停止 broker。

**验证**：
```powershell
netstat -ano | findstr :1883
# 应该看到 LISTENING 状态
```

### 虹膜凭证特殊说明

虹膜凭证下发时需要 **faceImage**（人脸照片 Base64）：
- IAMS 下发时**不包含** faceImage
- 数据库**不存储** faceImage
- 下发到虹膜设备时，后端**自己添加** faceImage
- 照片内容任意（卡通、工作卡片照片都可以）

详细说明参考：`更新说明-faceImage.md`

## 🐛 常见问题

### 1. MQTT Broker 启动失败：端口 1883 被占用

**错误**：
```
Error: listen EADDRINUSE: address already in use :::1883
```

**解决**：
```powershell
# 1. 查找占用进程
netstat -ano | findstr :1883

# 2. 停止占用进程（替换 PID）
Stop-Process -Id <PID> -Force

# 3. 重新启动 broker
.\start-mqtt.ps1
```

---

### 2. IAMS 显示"断开"

**原因**：MQTT broker 没有启动或连接失败。

**解决**：
1. 检查 broker 是否运行：`netstat -ano | findstr :1883`
2. 如果没有，启动 broker：`.\start-mqtt.ps1`
3. 重启诺亚后端

---

### 3. 设备列表显示"暂无设备"

**原因**：数据库刚重建，没有设备配置。

**解决**：重启诺亚后端，会自动初始化默认设备。

---

### 4. 下发记录为空

**原因**：还没有进行过凭证下发操作。

**解决**：
1. 调用模拟 IAMS 下发 API：`POST /api/devices/simulate-iams`
2. 或等待 MQTT 收到 IAMS 下发指令
3. 或点击"手动触发凭证下发"按钮

---

### 5. OpenClaw 崩溃

```powershell
# 1. 查看日志
Get-Content "C:\Users\ADMINI~1\AppData\Local\Temp\openclaw\openclaw-$(Get-Date -Format 'yyyy-MM-dd').log" -Tail 100

# 2. 重启
openclaw gateway restart
```

---

### 6. 掌纹设备连接失败

**原因**：掌纹设备服务未运行。

**解决**：
1. 检查 `maser_H01_wa04_V7.77.exe` 是否运行
2. 启动设备服务

---

### 7. 虹膜设备无照片返回

**原因**：`needImages` 参数未设为 1。

**解决**：确保请求中包含 `needImages=1`
