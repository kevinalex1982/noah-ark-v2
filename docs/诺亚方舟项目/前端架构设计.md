## 四、核心功能

### 4.1 倒计时功能（所有 kiosk 页面）
- **组件**：`components/kiosk/IdleTimer.tsx`
- **超时时间**：60 秒无操作返回待机页
- **显示方式**：前 30 秒静默，后 30 秒显示进度条
- **进度条颜色**：
  - 30-20 秒：蓝色
  - 20-10 秒：黄色
  - 10-0 秒：橙色
- **用户操作**：点击、移动、键盘输入都会重置倒计时

### 4.2 身份证验证
- **格式**：18 位（前 17 位数字 + 最后 1 位数字或 X）
- **软键盘**：0-9 数字键 + 清空 + 退格
- **API**：`/api/auth/verify-identity?identityId=xxx`
- **返回**：
  - success=true：身份证存在，返回 personId、personName
  - success=false：库中无此身份证信息（IDENTITY_NOT_FOUND）

### 4.3 认证方式显示
- **API**：`/api/auth/types?identityId=xxx`
- **动态显示**：根据用户实际凭证类型显示按钮
- **凭证类型**：
  - type=5：密码
  - type=7：虹膜
  - type=8：掌纹
  - type=9：胁迫码（不显示，系统自动识别）
- **测试数据**：
  - 111111111111111111（18 个 1）：密码 (5) + 虹膜 (7) + 胁迫码 (9)
  - 222222222222222222（18 个 2）：密码 (5) + 掌纹 (8) + 胁迫码 (9)

### 4.4 组合认证
- **动态步骤**：根据 authTypes 生成
- **排除胁迫码**：type=9 不显示在界面上
- **示例**：
  - 111111111111111111：密码 → 虹膜（2 步）
  - 222222222222222222：密码 → 掌纹（2 步）

### 4.5 虹膜认证 ⭐
- **设备要求**：虹膜设备需要人脸照片（faceImage）
- **前端处理**：无需处理，后端自动添加
- **显示**：扫描动画 + 倒计时

### 4.6 掌纹认证
- **设备**：掌纹/静脉设备
- **显示**：扫描动画 + 倒计时

| 8 | 掌纹 | 掌纹设备 ✅ |

- 客户端组件必须加 `'use client'`
- 避免 hydration 错误（时间只在客户端更新）
- CSS 中 `@import` 必须在文件第一行
