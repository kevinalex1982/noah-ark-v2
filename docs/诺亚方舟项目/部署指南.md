## 一、快速部署

### 方式一：直接运行（开发模式）

适合快速测试，无需打包：

```powershell
# 1. 启动后端服务
npm run dev

# 2. 另开终端，启动 Electron 客户端
cd electron && npm run dev
```

### 方式二：打包成安装程序

适合现场部署：

```powershell
# 需要能访问 GitHub（可能需要代理/VPN）
.\build-installer.ps1
```

**注意**：打包过程需要从 GitHub 下载工具，如果网络不通，请参考"网络问题解决"章节。

## 二、系统架构

### 1.1 组件说明

| 组件 | 说明 | 必须 |
|------|------|------|
| **Next.js 后端服务** | Web服务和API，端口3001 | ✅ 必须 |
| **Electron 客户端** | 桌面窗口，加载Next.js页面 | ✅ 必须 |
| **MQTT Broker** | 消息代理，连接 IAMS | ✅ 必须（IAMS通信） |
| **MQTT Client** | 测试客户端 | ❌ 可选（仅测试用） |

### 1.2 运行关系

```
┌─────────────────────────────────────────────────────────────┐
│                        现场环境                              │
│                                                             │

│  │   Electron  │      │         Next.js 服务            │  │

│  │             │      │   - Web页面                      │  │
│  └─────────────┘      │   - REST API                     │  │
│                       │   - 设备代理                      │  │
│                       │   - MQTT Client → IAMS           │  │
│                       └────────────┬────────────────────┘  │
│                                    │                        │
│                       ┌────────────┼────────────────────┐  │
│                       ↓            ↓                      │  │

                              │

                    ┌──────────────────┐
                    │   IAMS Broker    │
                    │ 58.33.106.19:3881│
```

**说明**：MQTT Client 内嵌在 Next.js 服务中，连接 IAMS Broker，用于接收凭证操作指令和上报事件。

### 1.3 架构说明

- **MQTT 连接**：连接 IAMS Broker（mqtt://58.33.106.19:3881），无需本地 MQTT Broker

## 二、部署方案

### 方案A：一体化部署（推荐）

**场景**：所有组件部署在同一台机器上

**优点**：
- 部署简单
- 无需修改配置
- 网络延迟最低

**结构**：
```
现场机器
├── noah-ark/                    # 主程序目录
│   ├── .next/                   # Next.js 编译产物
│   ├── electron/dist/           # Electron 编译产物
│   ├── data/                    # 开发模式数据目录
│   │   ├── noah-ark.db          # SQLite 数据库（开发用）
│   │   └── settings.json        # 系统配置（开发用）
│   ├── node_modules/            # 依赖
│   ├── package.json
│   └── start-production.ps1     # 启动脚本
│
└── AppData/Roaming/noah-ark-electron/  # 生产模式数据目录
    ├── noah-ark.db              # SQLite 数据库（更新不丢失）
    ├── settings.json            # 系统配置（更新不丢失）
    └── client-config.json       # 客户端配置
```

### 方案B：分离部署

**场景**：Next.js 服务和 Electron 客户端在不同机器

**需要修改**：
1. Electron 配置服务器地址
2. Next.js 允许跨域访问

## 三、打包步骤

### 3.1 Next.js 打包

```powershell
# 1. 安装依赖（首次）
npm install

# 2. 编译项目
npm run build

# 3. 测试生产版本
npm run start
```

**输出**：
- `.next/` 目录 - 编译产物
- `public/` 目录 - 静态资源

**注意**：`npm run build` 会生成 `.next/standalone/` 目录（如果配置了 `output: 'standalone'`），可以独立运行。

### 3.2 Electron 打包

**当前状态**：Electron 仅在开发模式运行

**打包方案**：

#### 方案1：使用 electron-builder（推荐）

```powershell
# 1. 安装 electron-builder
cd electron
npm install --save-dev electron-builder

# 2. 修改 electron/package.json
# 添加 build 配置

# 3. 打包
npm run build  # 编译 TypeScript
npx electron-builder --win
```

**输出**：
- `electron/dist/` - 可执行文件安装包

#### 方案2：手动打包

```powershell
# 1. 编译 TypeScript
cd electron
npm run build

# 2. 复制必要文件
# - dist/main.js
# - dist/preload.js
# - node_modules/
# - package.json
```

### 3.3 MQTT Broker

MQTT Broker 是一个 Node.js 脚本，不需要单独打包。

**启动方式**：
```powershell
node scripts/mqtt-broker.js
```

## 五、现场部署步骤

### 5.1 首次部署

#### 步骤1：准备环境

```powershell
# 检查 Node.js 版本（需要 18+）
node --version

# 如果没有安装，从 https://nodejs.org 下载安装
```

#### 步骤2：复制文件

将以下目录复制到现场机器：

```
noah-ark/
├── .next/
├── public/
├── app/
├── lib/
├── scripts/mqtt-broker.js
├── electron/dist/
├── package.json
├── next.config.ts
├── tailwind.config.ts
├── postcss.config.mjs
└── tsconfig.json

# 注意：生产模式数据存储在 AppData 目录，不需要复制
```

#### 步骤3：安装依赖

```powershell
cd noah-ark
npm install --production
```

#### 步骤4：配置系统

修改 `%APPDATA%\noah-ark-electron\settings.json`（如果文件不存在，首次启动时会自动创建）：

```json
{
  "authTimeout": 60,
  "successReturnTime": 10,
  "irisEndpoint": "http://192.168.3.202:9003",
  "palmEndpoint": "http://192.168.3.203:8080"
}
```

**重要**：根据现场实际设备IP修改 `irisEndpoint` 和 `palmEndpoint`

#### 步骤5：启动服务

**方式1：使用启动脚本**

```powershell
# 启动 Next.js 服务
.\start-production.ps1

# 另开终端，启动 MQTT Broker
.\start-mqtt.ps1

# 启动 Electron 客户端
.\electron\start-electron.ps1
```

**方式2：手动启动**

```powershell
# 终端1：启动 Next.js
npm run start

# 终端2：启动 MQTT Broker
node scripts/mqtt-broker.js

# 终端3：启动 Electron
cd electron
npm run start
```

### 5.2 日常启动

```powershell
# 1. 启动后端服务
npm run start

# 2. 启动 MQTT Broker
node scripts/mqtt-broker.js

# 3. 启动 Electron 客户端
cd electron && npm run start
```

## 六、生产启动脚本

创建 `start-production.ps1`：

```powershell
# 诺亚方舟 - 生产环境启动脚本

Write-Host "================================" -ForegroundColor Cyan
Write-Host "  诺亚方舟 - 生产服务" -ForegroundColor Cyan
Write-Host "================================" -ForegroundColor Cyan

Set-Location $PSScriptRoot

# 检查 Node.js
if (-not (Get-Command node -ErrorAction SilentlyContinue)) {
    Write-Host "错误：未安装 Node.js" -ForegroundColor Red
    exit 1
}

# 启动 Next.js 生产服务
Write-Host "启动 Next.js 服务 (端口 3001)..." -ForegroundColor Yellow
node_modules\.bin\next start -p 3001
```

创建 `start-all.ps1`（一键启动）：

```powershell
# 诺亚方舟 - 一键启动

Write-Host "启动所有服务..." -ForegroundColor Cyan

# 启动 Next.js（后台）
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PSScriptRoot'; npm run start"

# 等待2秒
Start-Sleep -Seconds 2

# 启动 MQTT Broker（后台）
Start-Process powershell -ArgumentList "-NoExit", "-Command", "cd '$PSScriptRoot'; node scripts/mqtt-broker.js"

# 等待1秒
Start-Sleep -Seconds 1

# 启动 Electron
Start-Process powershell -ArgumentList "-Command", "cd '$PSScriptRoot\electron'; npm run start"

Write-Host "所有服务已启动" -ForegroundColor Green
```

## 八、常见问题

### Q1: Electron 显示"无法连接到服务器"

**原因**：Next.js 服务未启动

**解决**：
```powershell
# 先启动 Next.js
npm run start
```

### Q2: MQTT 连接失败

**原因**：MQTT Broker 未启动

**解决**：
```powershell
node scripts/mqtt-broker.js
```

### Q3: 虹膜/掌纹设备连接失败

**原因**：设备地址配置错误

**解决**：
1. 打开 `%APPDATA%\noah-ark-electron\settings.json`
2. 修改 `irisEndpoint` 或 `palmEndpoint` 为正确的设备IP
3. 重启 Next.js 服务

### Q4: 端口被占用

**检查端口占用**：
```powershell
netstat -ano | findstr :3001
netstat -ano | findstr :1883
```

**结束占用进程**：
```powershell
# 找到 PID 后
taskkill /PID <PID> /F
```

## 十三、网络问题解决（打包失败）

### 问题描述

打包时遇到以下错误：
```
Get "https://github.com/electron-userland/electron-builder-binaries/releases/download/winCodeSign-2.6.0/winCodeSign-2.6.0.7z": 
dial tcp 20.205.243.166:443: connectex: A connection attempt failed
```

**原因**：electron-builder 需要从 GitHub 下载工具（winCodeSign、rcedit 等），国内网络可能无法访问。

### 解决方案

#### 方案1：使用代理/VPN

开启代理后执行打包命令。

#### 方案2：手动下载工具

1. 从能访问 GitHub 的环境下载以下文件：
   - https://github.com/electron-userland/electron-builder-binaries/releases/download/winCodeSign-2.6.0/winCodeSign-2.6.0.7z
   - https://github.com/electron-userland/electron-builder-binaries/releases/download/rcedit-2.0.1/rcedit-2.0.1.7z

2. 解压到本地缓存目录：
   ```
   %LOCALAPPDATA%\electron-builder\Cache\winCodeSign\
   %LOCALAPPDATA%\electron-builder\Cache\rcedit\
   ```

3. 重新执行打包命令

#### 方案3：只打包目录（不生成安装程序）

适合内部使用，不生成 exe 安装程序：

```powershell
cd electron
npm run build
# 生成的文件在 release/win-unpacked/ 目录
```

### 打包后的目录结构

```
electron/release/win-unpacked/

├── resources/
```

## 十五、数据目录迁移到 AppData（2026-04-12 重要）

### 15.1 背景

早期版本中，所有数据文件存储在项目的 `data/` 目录下。打包安装时，安装程序会覆盖安装目录，导致 `data/` 下的数据库和配置文件被清空，用户数据丢失。

### 15.2 迁移方案

所有持久化数据已迁移到 Electron 的用户数据目录（Windows: `AppData/Roaming/noah-ark-electron/`），该目录不会被安装程序覆盖。

**迁移后的数据位置：**

| 数据 | 旧路径（已废弃） | 新路径 |
|------|-----------------|--------|
| 数据库 | `安装目录/resources/app/data/noah-ark.db` | `%APPDATA%\noah-ark-electron\noah-ark.db` |
| 系统设置 | `安装目录/resources/app/data/settings.json` | `%APPDATA%\noah-ark-electron\settings.json` |
| 客户端配置 | `%APPDATA%\noah-ark-electron\client-config.json` | （不变，已在此） |
| MQTT 日志 | `安装目录/resources/app/data/mqtt_logs/` | `%APPDATA%\noah-ark-electron\mqtt_logs\` |
| MQTT 事件 | `安装目录/resources/app/data/mqttevent.json` | `%APPDATA%\noah-ark-electron\mqttevent.json` |
| MQTT 下发消息 | `安装目录/resources/app/data/mqtt-log-reply/` | `%APPDATA%\noah-ark-electron\mqtt-log-reply\` |
| 人脸照片样本 | `安装目录/resources/app/data/face_photo_sample.txt` | `%APPDATA%\noah-ark-electron\face_photo_sample.txt` |

**快速访问数据目录：**
```powershell
# 在文件资源管理器地址栏输入
%APPDATA%\noah-ark-electron\
# 或
C:\Users\用户名\AppData\Roaming\noah-ark-electron\
```

### 15.3 实现机制

- **`electron/main.ts`**：启动 Next.js 时设置 `DATA_DIR` 环境变量 = `app.getPath('userData')`
- **`lib/database.ts`**：优先读取 `DATABASE_PATH` 环境变量，其次使用 `DATA_DIR`
- **`lib/settings.ts`**：优先读取 `DATA_DIR` 环境变量，fallback 到项目 `data/` 目录
- **开发模式**：未设置环境变量时自动使用项目根目录的 `data/` 目录（不影响开发）
- **生产模式**：Electron 自动设置环境变量，指向 AppData

### 15.4 打包时不再包含 data 目录

`electron/package.json` 的 `extraResources` 已移除 `../data` 目录的打包引用。安装包体积大幅减小（不再打包 4GB+ 的数据库文件）。

### 15.5 更新流程（覆盖安装，数据不丢失）

**当需要更新程序时：**

```
1. 下载新的安装包（.exe）
2. 运行安装程序，覆盖安装到相同目录
   → 安装目录的代码被更新
   → AppData 的数据完全不受影响 ✅
3. 启动应用
   → 自动读取 AppData 中的数据库和配置
   → 所有凭证、设置、日志保持原样
4. 如需更新配置，通过系统设置页面修改即可
```

### 15.6 数据备份

**备份：** 直接复制 `%APPDATA%\noah-ark-electron\` 整个目录到其他位置

**恢复：** 将备份目录中的文件复制回 `%APPDATA%\noah-ark-electron\`

**关键文件：**
- `noah-ark.db` — 数据库（最重要，包含所有凭证和记录）
- `settings.json` — 系统配置（设备地址、MQTT 连接等）
- `client-config.json` — 客户端配置（服务器地址）

### 15.7 注意事项

1. **首次安装后数据为空** — 新安装的 AppData 目录下没有数据库文件，应用启动时会自动创建。需要通过 IAMS 平台下发凭证，或通过模拟下发添加测试数据。
2. **卸载程序不删除数据** — 卸载时 AppData 目录保留，重新安装后数据仍在。如需彻底清除，手动删除 `%APPDATA%\noah-ark-electron\` 目录。
3. **旧版本迁移** — 如果从旧版本升级（数据在 `data/` 目录下），需要手动将 `data/noah-ark.db` 和 `data/settings.json` 复制到 `%APPDATA%\noah-ark-electron\`。
4. **开发模式不受影响** — 开发时（`npm run dev`）数据仍在项目 `data/` 目录中。
