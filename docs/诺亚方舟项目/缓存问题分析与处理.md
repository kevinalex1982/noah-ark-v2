# 缓存问题分析与处理

> 创建时间：2026-04-18
> 用途：帮助现场排查和解决缓存导致的前端显示异常问题

## 系统中的缓存类型

### 1. Next.js 构建缓存

| 项目 | 说明 |
|------|------|
| **位置** | `.next/cache/` 目录 |
| **内容** | Turbopack 编译缓存、Webpack 模块缓存 |
| **影响** | 新代码改动后如果缓存未更新，服务端返回的可能是旧版本 JS |
| **清理方式** | 删除 `.next/cache/` 整个目录 |
| **安全** | 清理不影响用户数据（settings.json、noah-ark.db 等） |
| **何时清理** | 每次打包时已自动排除；前端代码更新后可手动清理 |

### 2. 浏览器（Chromium）缓存

| 项目 | 说明 |
|------|------|
| **位置** | Electron 窗口 session 缓存（内存中） |
| **内容** | HTTP 响应缓存、CSS/JS 文件缓存、Service Worker 缓存 |
| **影响** | 浏览器可能加载旧的 JS chunk 文件 |
| **清理方式** | `mainWindow.webContents.session.clearCache()` |
| **安全** | 清理不影响用户数据 |
| **何时清理** | 页面显示异常、样式错乱、按钮缺失时 |

### 3. Electron 本地存储

| 项目 | 说明 |
|------|------|
| **位置** | `AppData/Roaming/noah-ark-electron/` |
| **内容** | `settings.json`、`noah-ark.db`、`client-config.json`、日志文件 |
| **注意** | 这些是用户数据，**不是缓存**，不应随意删除 |
| **哪些可以删** | `logs/` 目录下的日志文件可以安全删除 |
| **哪些不能删** | `settings.json`（设备配置）、`noah-ark.db`（数据库）、`client-config.json`（服务器地址） |

### 4. IAMS Payload 缓存

| 项目 | 说明 |
|------|------|
| **位置** | `{DATA_DIR}/iams_payloads/` |
| **内容** | IAMS 下发的虹膜/掌纹完整数据（JSON 文件） |
| **上限** | 最多保留 10 个文件，自动清理旧文件 |
| **作用** | 用于排查 IAMS 指令下发问题 |

## 现场排查步骤

### 第一步：清理缓存并刷新

1. 进入 **系统设置** 页面
2. 点击 **清理缓存** 按钮
3. 等待清理完成提示
4. 点击 **刷新页面** 按钮
5. 检查凭证管理页面是否恢复正常

### 第二步：检查页面版本

打开浏览器开发者工具（Electron 托盘菜单 → 打开开发者工具），在 Console 中输入：

```javascript
document.querySelectorAll('script[src]').forEach(s => console.log(s.src));
```

查看加载的 JS 文件路径。Next.js 生产构建的 chunk 文件名带有内容哈希（如 `9a3b2c1d.js`），如果文件名和预期不同，说明加载的是旧版本。

### 第三步：检查 `.next` 目录

如果安装后仍有问题，检查安装目录下 `.next/server/` 中的 JS 文件是否是最新版本。

## 打包时缓存处理

`electron/package.json` 中 extraResources 已配置排除缓存：

```json
{
  "from": "../.next",
  "to": "app/.next",
  "filter": [
    "**/*",
    "!dev/**/*",
    "!.next/cache/**/*"
  ]
}
```

这确保安装包中不包含旧的 Next.js 缓存。

## Electron 启动时缓存清理

打包后首次启动时，`cleanNextCache()` 会自动清理 `.next/cache/` 目录。这是双重保险。
