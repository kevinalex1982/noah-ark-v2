## 一、虹膜设备解析

### 1.1 设备信息

| 项目 | 值 |
|------|-----|
| 设备类型 | 虹膜识别设备 |
| 设备 IP | 192.168.3.202 |
| 设备端口 | 9003 |
| 设备 SN | 200527-J80-S00-0039 |
| 接口协议 | HTTP POST |
| 原始协议 | `iris_protocol.txt` |

### 1.2 接口解析

**协议文档接口**：获取人员信息接口 (`/members`)

**实际请求**：
```
POST http://192.168.3.202:9003/members
Content-Type: application/json

{
  "count": 10,
  "key": "",
  "lastStaffNumDec": "",
  "needImages": 1
}
```

**⚠️ 关键发现**：
- `needImages` 参数**必须设为 1**，否则不会返回虹膜照片
- 虹膜图片是 **BMP 格式的 Base64 编码**
- 如果只注册了一只眼睛，可能只有 `irisLeftImage` 或 `irisRightImage` 其中一个有值

### 1.3 下发人员到虹膜设备

**接口**：`/members` (POST)

**请求示例**：
```json
{
  "staffNumDec": "111112345",
  "staffName": "test1",
  "idType": 0,
  "idNum": "111111111111111111",
  "featureType": 3,
  "featureData": "base64_encoded_iris_feature",
  "faceImage": "base64_encoded_face_photo",
  "faceImageName": "face_111112345"
}
```

**字段说明**：
- `staffNumDec`: 用户 ID（长数字格式）
- `staffName`: 用户姓名
- `featureType`: 特征类型，3=虹膜
- `featureData`: 虹膜特征数据（Base64）
- `faceImage`: **人脸照片 Base64**（虹膜设备要求）
- `faceImageName`: 人脸照片文件名

**faceImage 重要说明**：
- **用途**：虹膜设备要求的人脸照片，用于显示用户信息
- **内容**：任意照片即可（卡通、工作卡片照片都可以）
- **验证**：虹膜设备**不会验证**照片是否包含人脸
- **来源**：
  - IAMS 下发时**不包含** faceImage
  - 数据库**不存储** faceImage
  - 下发到虹膜设备时，后端**自己添加** faceImage
- **示例代码**：参考 `更新说明-faceImage.md`

**响应示例**：
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "staffNumDec": "111112345",
    "staffName": "test1"
  }
}
```

## 二、掌纹设备解析

### 2.1 设备信息

| 项目 | 值 |
|------|-----|
| 设备类型 | 掌纹/静脉识别设备 |
| 设备 IP | 127.0.0.1 |
| 设备端口 | 8080 |
| 服务程序 | `maser_H01_wa04_V7.77.exe` |
| 接口协议 | HTTP GET/POST |
| 原始协议 | `掌纹设备文档.txt` |

### 2.2 接口清单

| 接口编号 | 功能 | 说明 |
|----------|------|------|
| 105 | 查询所有用户 | 返回设备中已注册的用户列表 |
| 106 | 添加用户 | 下发用户特征到设备 |
| 107 | 删除全部用户 | 清空设备所有用户 |
| 108 | 删除用户 | 删除设备中的用户 |
| 109 | 获取用户特征 | 查询指定用户的特征数据 |
| 110 | 下发用户特征 | 同 106，添加用户 |

### 2.3 ⚠️ 关键注意事项

1. **必须使用 `http.client` 模块**，不能用 `requests`
2. **sendData 参数必须放在 URL 中**，不能作为 POST body
3. **JSON 不能有空格**（会导致解析失败）
4. 服务程序必须运行：`maser_H01_wa04_V7.77.exe`
5. **sendData 不能 URL 编码**，直接传原始 JSON 字符串

### 2.4 🔑 featureData 结构与 userId 匹配规则

**featureData 结构**：
```
[Base64特征数据]=[用户名]^^^^^^^^...
```

**示例**：
```
...AAAAA=TgOd/m3A23xiH5AAAAAAAAAAAAAneo^^^^^^^^...
                                    ↑
                              用户名隐藏在这里
```

**⚠️ 重要规则**：
- 109 接口返回的 featureData 末尾包含用户名
- **110 下发时的 userId 必须和 featureData 里的用户名匹配**
- 不匹配时设备返回 `code: 404`

**正确下发流程**：
```
1. 109 获取特征 → 得到 featureData
2. 提取用户名  → 从 featureData 末尾提取（= 后面，^ 前面）
3. 110 下发    → 使用相同的 userId
```

**代码示例**：
```javascript
// 从 featureData 末尾提取 userId
function extractUserId(featureData) {
  const lastEq = featureData.lastIndexOf('=');
  const afterEq = featureData.substring(lastEq + 1);
  return afterEq.replace(/\^+$/, '').trim();  // 移除末尾的 ^
}

// 109 返回
const featureData = "...AAAAA=neo^^^^^...";

// 提取并下发
const userId = extractUserId(featureData);  // "neo"
// 下发时必须用 "neo"，不能用其他 userId
```

### 2.5 Python 示例代码

```python
import http.client
import json

host = '127.0.0.1'
port = 8080
conn = http.client.HTTPConnection(host, port, timeout=5)

# sendData 放在 URL 中（JSON 不能有空格！）
send_data = '{"request":"106","userId":"test2","featureData":"2eHwPbSb..."}'
url = f'/api?sendData={send_data}'

conn.request('POST', url)
response = conn.getresponse()
result = json.loads(response.read().decode('utf-8'))
conn.close()

print(result)
```

### 2.5 接口详细说明

#### 105 接口 - 查询所有用户

**请求**：
```
POST http://127.0.0.1:8080/api?sendData={"request":"105"}
```

**响应**：
```json
{
  "response": "105",
  "userNumber": 1,
  "userData": [{"userId": "cjj"}]
}
```

#### 106/110 接口 - 添加用户

**请求**：
```
POST http://127.0.0.1:8080/api?sendData={"request":"110","userId":"neo","featureData":"2eHwPbSb..."}
```

**⚠️ 重要发现**：
- featureData 末尾包含 userId（格式：`...=用户名^^^^^...`）
- **110 下发的 userId 必须和 featureData 里的 userId 匹配**
- 如果不匹配，设备返回 `code: 404`

**正确流程**：
1. 用 109 获取用户特征
2. 从 featureData 末尾提取 userId
3. 用相同的 userId 调用 110 下发

**响应**：
```json
{
  "response": "110",
  "code": "200"
}
```

#### 108 接口 - 删除用户

**请求**：
```
POST http://127.0.0.1:8080/api?sendData={"request":"108","userId":"test2"}
```

**响应**：
```json
{
  "response": "108",
  "code": "200",
  "des": "删除成功"
}
```

#### 109 接口 - 获取用户特征

**请求**：
```
POST http://127.0.0.1:8080/api?sendData={"request":"109","userId":"test2"}
```

**响应**：
```json
{
  "response": "109",
  "code": "200",
  "des": "userId:test2;featureData:2eHwPbSb..."
}
```

## 三、设备对接常见问题

### Q1: 掌纹设备连接失败？
**A**: 检查 `maser_H01_wa04_V7.77.exe` 是否运行

### Q2: 虹膜设备无照片返回？
**A**: 确保 `needImages=1`

### Q3: 掌纹设备返回 JSON 解析失败？
**A**: sendData 中的 JSON 不能有空格

### Q4: 使用 requests 模块报错？
**A**: 必须使用 `http.client` 模块，不能用 `requests`

### Q5: faceImage 应该是什么内容？
**A**: 任意 Base64 编码的照片（卡通、工作卡片照片都可以），虹膜设备不会验证是否有人脸

### Q6: faceImage 从哪里来？
**A**: IAMS 下发时没有，数据库也不存，后端在下发到虹膜设备时自己添加
