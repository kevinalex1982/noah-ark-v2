## 一、需求分析

### 1.1 IAMS 协议核心字段

根据 IAMS 基础协议（诺亚修改版），凭证同步包含以下关键字段：

```json
{
  "time": 1705371902981,
  "requestId": "req-xxx",
  "deviceId": "iris-device-001",
  "op": "passport-add",
  "data": {
    "personId": "111111111111111111",    // 用户编码（18位）
    "passportVer": "iris-device-001-xxx", // 凭证库版本号
    "id": 998,                            // 凭证唯一 ID
    "type": 7,                            // 识别类型：7=虹膜，8=掌纹
    "authModel": 2,                       // 识别模式
    "authTypeList": "5,7,9",             // 识别凭证类型列表
    "showInfo": ["欢迎", "张三"],         // 显示信息
    "tags": [1, 2]                        // 标签ID列表
  }
}
```

### 1.2 用户编码说明

**重要约定**：
- `person_id` 字段统一作为用户编码（18位）
- 系统界面统一显示为**"用户编码"**
- 已移除 `id_card` 字段，不再单独存储

**测试固定值**：
| 设备类型 | 用户编码（person_id） | 说明 |
|----------|----------------------|------|
| 虹膜设备 | `111111111111111111` | 18个1 |
| 掌纹设备 | `222222222222222222` | 18个2 |

### 1.3 ⭐ 凭证库版本号机制（重要！）

**协议原文**：
> "每次执行凭证操作，设备需要检验凭证库版本号与当前设备上存的版本号是否相同。**不相同时不执行操作**。"

**说明**：
- passportVer 是 IAMS 协议字段，用于设备和 IAMS 平台之间的版本同步
- 本地数据库**不需要存储**passportVer
- 每次收到 IAMS 的 passport-add 请求时，直接使用请求中的 passportVer 下发到设备
- 设备自己维护版本号，本地系统不干预

## 二、数据库设计

### 2.1 表结构总览

```
┌─────────────────────────┐
│   device_config         │  ← 设备配置表（仅设备信息）
├─────────────────────────┤

│   auth_log              │  ← 认证记录表（通行记录）
```

### 2.2 设备配置表 (device_config)

**用途**：存储设备配置信息（设备 ID、名称、类型、通信地址等）

### 2.3 凭证表 (credentials)

```sql
CREATE TABLE credentials (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    
    -- ========== 人员信息 ==========
    person_id VARCHAR(64) NOT NULL COMMENT '用户编码（18位）',
    person_name VARCHAR(100) COMMENT '人员姓名',
    person_type CHAR(1) DEFAULT 'n' COMMENT '人员类型：n=普通，v=访客',
    
    -- ========== 凭证信息 ==========
    credential_id BIGINT NOT NULL UNIQUE COMMENT '凭证唯一 ID（IAMS id）',
    type TINYINT NOT NULL COMMENT '识别类型：5=密码，7=虹膜，8=掌纹，9=胁迫码',
    content TEXT COMMENT '凭证内容（密码/虹膜数据等）',
    
    -- 虹膜专用字段（type=7 时使用）
    iris_left_image TEXT COMMENT '左眼虹膜 Base64',
    iris_right_image TEXT COMMENT '右眼虹膜 Base64',
    
    -- 掌纹专用字段（type=8 时使用）
    palm_feature TEXT COMMENT '掌纹特征 Base64',
    custom_id VARCHAR(128) COMMENT '自定义ID（掌纹设备userId等）',
    
    -- ========== 显示和标签 ==========
    show_info TEXT COMMENT '显示信息（鉴权成功时屏幕展示文本）',
    tags VARCHAR(255) COMMENT '标签ID列表',
    
    -- ========== 认证策略 ==========
    auth_model TINYINT DEFAULT 1 COMMENT '识别模式：1=单凭证，2=多凭证组合',
    auth_type_list VARCHAR(64) COMMENT '凭证类型列表（如"5,7,9"）',
    box_list VARCHAR(255) COMMENT '用户箱号列表（如"rc10|rc20"）',
    
    -- ========== 其他字段 ==========
    enable TINYINT DEFAULT 1 COMMENT '启用状态：1=启用，2=禁用',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    -- ========== 索引 ==========
    INDEX idx_person_id (person_id),
    INDEX idx_type (type),
    INDEX idx_credential_id (credential_id)
);
```

### 2.4 认证记录表 (auth_log)

**用途**：记录所有认证通行事件，用于上传 IAMS 平台（pass-log）

```sql
CREATE TABLE auth_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    
    -- ========== 认证基本信息 ==========
    log_id VARCHAR(64) NOT NULL UNIQUE COMMENT '日志唯一 ID（UUID）',
    person_id VARCHAR(64) COMMENT '用户编码（未识别时为空）',
    credential_id BIGINT COMMENT '凭证 ID（未识别时为 -1）',
    
    -- ========== 认证详情 ==========
    auth_type TINYINT NOT NULL COMMENT '认证类型：1=人脸，5=密码，7=虹膜，8=掌纹，9=胁迫码',
    auth_result TINYINT NOT NULL COMMENT '认证结果：1=成功，2=失败',
    fail_type TINYINT DEFAULT 0 COMMENT '失败类型：0=成功，1=未注册，2=失效，3=规则限制...',
    fail_msg VARCHAR(255) COMMENT '失败详情',
    
    -- ========== 生物识别特有字段 ==========
    similarity DECIMAL(5,2) COMMENT '相似度（百分比，如 85.5）',
    threshold DECIMAL(5,2) COMMENT '阈值（百分比，如 70）',
    captured_img TEXT COMMENT '抓拍照片 Base64',
    
    -- ========== 设备信息 ==========
    device_id VARCHAR(64) NOT NULL COMMENT '设备 ID',
    auth_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '认证时间',
    
    -- ========== IAMS 上传状态 ==========
    upload_status VARCHAR(16) DEFAULT 'pending' COMMENT '上传状态：pending/uploaded/failed',
    upload_time DATETIME COMMENT '上传时间',
    upload_response TEXT COMMENT 'IAMS 响应',
    
    -- ========== 系统字段 ==========
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    -- ========== 索引 ==========
    INDEX idx_person_id (person_id),
    INDEX idx_auth_time (auth_time),
    INDEX idx_upload_status (upload_status)
);
```

```sql
    INDEX idx_status (status),
    INDEX idx_device_id (device_id)

```

## 三、版本号处理说明

### 3.1 passportVer 的处理方式

**核心原则**：本地数据库**不存储**passportVer

**原因**：
1. passportVer 是 IAMS 平台和设备之间的版本标识
2. 本地系统只是中转，不需要维护版本号
3. 设备自己维护版本号，并与 IAMS 平台同步

**处理流程**：
```
IAMS 平台 → passport-add (含 passportVer) → 本地系统
                                               ↓
本地系统 → passport-add (含相同 passportVer) → 设备
                                               ↓
设备校验 passportVer → 执行操作
```

### 3.2 passport-add 下发流程

```typescript
async function handlePassportAddFromIAMS(iamsMessage: any) {
  // 1. 从 IAMS 消息中提取 passportVer
  const passportVer = iamsMessage.data.passportVer;
  
  // 2. 存储凭证到数据库（不存储 passportVer）
  await upsertCredential({
    person_id: iamsMessage.data.personId,
    person_name: iamsMessage.data.personName,
    credential_id: iamsMessage.data.id,
    type: iamsMessage.data.type,
    show_info: iamsMessage.data.showInfo?.join('|'),
    tags: iamsMessage.data.tags?.join(','),
    // ... 其他字段
  });
  
  // 3. 添加到同步队列，下发到设备（包含 passportVer）
  await addToSyncQueue({
    device_id: 'iris-device-001',
    action: 'passport-add',
    credential_id: iamsMessage.data.id,
    payload: {
      ...iamsMessage.data,
      passportVer: passportVer,  // ⭐ 直接透传给设备
    }
  });
}
```

### 3.3 设备端版本号校验

**设备端逻辑**（由设备固件实现，本地系统不干预）：
```
设备收到 passport-add
    ↓
提取 passportVer
    ↓
与本地存储的版本号对比
    ↓
相同 → 执行操作，更新版本号
不同 → 拒绝操作，返回错误
```

## 四、认证记录上传流程（pass-log）

### 4.1 认证完成后记录日志

```typescript
// TypeScript 示例
async function recordAuthLog(data: {
  personId: string;
  credentialId: number;
  authType: number;
  authResult: number;
  deviceId: string;
  similarity?: number;
  capturedImg?: string;
}) {
  const db = getDatabase();
  const logId = `log_${Date.now()}_${Math.random().toString(36).slice(2)}`;
  
  await db.execute({
    sql: `INSERT INTO auth_log (
      log_id, person_id, credential_id, auth_type, auth_result,
      device_id, similarity, captured_img, auth_time
    ) VALUES (?, ?, ?, ?, ?, ?, ?, ?, CURRENT_TIMESTAMP)`,
    args: [
      logId,
      data.personId,
      data.credentialId,
      data.authType,
      data.authResult,
      data.deviceId,
      data.similarity || null,
      data.capturedImg || null
    ]
  });
}
```

### 4.2 上传 IAMS pass-log

```typescript
// 上传待上传的认证记录
async function uploadAuthLogsToIAMS() {
  const db = getDatabase();
  
  // 查询待上传的记录
  const result = await db.execute({
    sql: `SELECT * FROM auth_log WHERE upload_status = 'pending' LIMIT 100`,
    args: []
  });
  
  for (const row of result.rows) {
    const log = row as AuthLog;
    
    // 构造 IAMS pass-log 消息
    const message = {
      time: Date.now(),
      requestId: `req_${Date.now()}`,
      deviceId: log.device_id,
      op: 'pass-log',
      data: {
        id: log.id,
        time: new Date(log.auth_time).getTime(),
        passportId: log.credential_id || -1,
        authType: log.auth_type,
        authResult: log.auth_result,
        failType: log.fail_type || 0,
        failMsg: log.fail_msg || '',
        similarity: log.similarity?.toString() || '',
        threshold: '70',
        capturedImg: log.captured_img || '',
        personName: log.person_name || ''
      }
    };
    
    // 发送到 IAMS
    const response = await publishToIAMS(message);
    
    // 更新上传状态
    await db.execute({
      sql: `UPDATE auth_log 
        SET upload_status = ?, upload_time = CURRENT_TIMESTAMP, upload_response = ?
        WHERE id = ?`,
      args: [response.success ? 'uploaded' : 'failed', JSON.stringify(response), log.id]
    });
  }
}
```

## 六、设计说明

### 6.1 数据库选择

- **SQLite** (@libsql/client) - 轻量级，适合本地部署
- **自动持久化** - 无需手动保存
- **兼容 Next.js** - 支持 Edge Runtime

### 6.2 表关系

```
credentials (凭证表)
    ↓ (person_id 关联)
auth_log (认证记录表)

device_config (设备配置表)
    ↓ (device_id 关联)
sync_queue (同步队列)
    ↓ (queue_id 关联)
sync_logs (同步日志)
```

### 6.3 索引设计
