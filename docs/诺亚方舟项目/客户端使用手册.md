## 1. 系统概述

诺亚宝库是一套生物识别设备管理系统，用于管理和控制虹膜、掌纹等生物识别设备，为用户提供多模态身份认证服务。系统由以下部分组成：

- **认证终端**：触屏操作界面，用于身份认证
- **管理后台**：设备管理、凭证管理、日志查看和系统配置
- **MQTT 通信**：与 IAMS 平台通信，接收凭证下发指令并上报通行记录
- **生物识别设备**：虹膜识别设备、掌纹识别设备

## 2. 安装与启动

### 2.1 安装

1. 双击安装程序 `诺亚宝库 Setup 1.0.0.exe`
2. 选择安装路径，点击"安装"
3. 安装完成后，程序自动启动

### 2.2 数据存储位置

系统所有持久化数据存储在用户数据目录（Windows: `%APPDATA%\noah-ark-electron\`），包括：

| 数据 | 文件 |
|------|------|
| 数据库 | `noah-ark.db` |
| 系统设置 | `settings.json` |
| MQTT 日志 | `mqtt_logs/` |
| MQTT 事件 | `mqttevent.json` |
| MQTT 下发消息 | `mqtt-log-reply/` |
| 人脸照片样本 | `face_photo_sample.txt` |

> **覆盖安装不会丢失数据**，因为数据存放在 AppData 目录而非安装目录。

### 2.3 启动

- 安装完成后程序自动启动
- 如需手动启动，从桌面快捷方式或开始菜单打开"诺亚宝库"
- 启动后系统托盘会出现应用图标

## 4. 认证终端（认证端）

认证终端供被认证人员使用，通过触屏完成身份验证。整个认证流程为：

```
待机首页 → 身份确认 → 选择认证方式 → 具体认证 → 认证成功/失败 → 返回待机
```

### 4.1 身份确认

**【截图：身份确认页面】**

此页面要求输入身份编码。

| 元素 | 说明 |
|------|------|
| 倒计时器 | 显示无人操作的剩余时间，超时自动返回首页 |
| 输入框 | 显示已输入的身份编码（最多 32 位） |
| 数字键盘 | 0-9 数字键，支持点击输入 |
| 清空按钮 | 清空当前输入 |
| 退格键 | 删除最后一位 |
| 下一步按钮 | 输入非空后可点击 |

**操作流程**：

1. 使用数字键盘输入身份编码
2. 点击"下一步"
3. 系统查询数据库验证编码是否存在
4. 验证通过 → 进入[选择认证方式](#42-选择认证方式)
5. 验证失败 → 提示"库中无此用户编码信息"

> 身份编码不一定是身份证号，长度限制为 1-32 位。

### 4.2 选择认证方式

**【截图：选择认证方式页面】**

系统根据该用户已录入的凭证类型，显示可用的认证方式。

| 认证方式 | 显示条件 |
|----------|----------|
| 密码认证 | 用户已录入密码凭证 |
| 虹膜认证 | 用户已录入虹膜凭证 |
| 掌纹认证 | 用户已录入掌纹凭证 |
| 组合认证 | 用户拥有 2 种或以上凭证类型 |

**操作**：点击对应的认证方式卡片进入相应的认证页面。也可点击"返回上一步"回到身份确认页。

### 4.3 密码认证

**【截图：密码认证页面】**

| 元素 | 说明 |
|------|------|
| 倒计时器 | 无人操作超时自动返回 |
| 密码输入框 | 以掩码形式显示 |
| 数字键盘 | 0-9 数字键、清空、退格 |
| 返回按钮 | 返回选择认证方式页 |
| 确认按钮 | 输入至少 4 位后可点击 |

**操作流程**：

1. 使用数字键盘输入密码（至少 4 位）
2. 点击"确认"
3. 系统验证密码，通过后显示认证成功页

### 4.4 虹膜认证

**【截图：虹膜认证页面】**

| 元素 | 说明 |
|------|------|
| 倒计时器 | 认证超时倒计时（默认 60 秒，可在系统设置中调整） |
| 虹膜图标 | 认证中时旋转动画，成功时显示绿色对勾 |
| 状态文字 | 显示当前状态（等待中 / 扫描中 / 成功 / 超时） |
| 提示信息 | "请保持眼睛与摄像头平视，不要眨眼" |
| 返回按钮 | 返回选择认证方式页 |
| 重试按钮 | 重新发起认证 |

**操作流程**：

1. 页面自动进入扫描状态
2. 注视虹膜摄像头，保持头部稳定
3. 识别成功后自动跳转到认证成功页
4. 若识别到其他人，会提示"识别到其他人，请等待您本人识别"
5. 超时未识别则显示超时提示，可点击"重试"

### 4.5 掌纹认证

**【截图：掌纹认证页面】**

| 元素 | 说明 |
|------|------|
| 倒计时器 | 认证超时倒计时 |
| 掌纹图标 | 认证中时脉冲动画 |
| 状态文字 | 等待中 / 扫描中 / 成功 / 超时 |
| 提示信息 | "请手掌平放，手指自然张开" |
| 返回按钮 | 返回选择认证方式页 |
| 重试按钮 | 重新发起认证 |

**操作流程**：

1. 页面自动进入扫描状态
2. 将手掌平放于扫描仪，手指自然张开
3. 保持手掌稳定等待识别
4. 识别成功后自动跳转到认证成功页
5. 超时可点击"重试"

### 4.6 组合认证

**【截图：组合认证页面】**

当用户拥有多种凭证类型时，可选择组合认证。系统会依次要求完成每一步认证。

| 元素 | 说明 |
|------|------|
| 进度指示器 | 圆形步骤条，已完成显示绿色对勾，当前步骤高亮闪烁 |
| 当前步骤 | 显示"步骤 X / Y"和当前认证类型名称 |
| 密码输入区 | 与独立密码认证相同的数字键盘 |
| 虹膜扫描区 | 与独立虹膜认证相同的扫描动画 |
| 掌纹扫描区 | 与独立掌纹认证相同的扫描动画 |
| 返回/上一步按钮 | 第一步时返回选择页，后续步骤返回上一步 |

**操作流程**：

1. 第一步为密码认证（如果用户有密码凭证），输入密码后点击"下一步"
2. 后续步骤依次进行虹膜或掌纹认证
3. 所有步骤完成后跳转到认证成功页
4. 任一步骤可返回上一步

### 4.7 认证成功 / 失败

**【截图：认证成功页面】**

| 元素 | 说明 |
|------|------|
| 图标 | 绿色对勾（成功）或红色叉号（失败） |
| 标题 | "认证成功" 或 "认证失败" |
| 欢迎信息 | 显示用户姓名（成功时） |
| 箱号信息 | 显示该用户对应的箱号（成功时） |
| 倒计时 | 自动返回待机页的倒计时（默认 10 秒，可设置） |
| 返回待机按钮 | 立即返回待机页 |

认证成功后，系统会自动上传通行记录到 IAMS 平台。

## 5. 管理后台

管理后台供管理员使用，提供设备管理、凭证查看、系统配置等功能。

**进入方式**：从待机首页无法直接访问，可通过浏览器访问 `/dashboard/devices` 或通过客户端内置入口进入。

管理后台包含 5 个子页面，顶部导航栏可在各页面之间切换。

### 5.1 设备管理

**【截图：设备管理页面】**

此页面展示系统运行状态和设备连接情况。

**状态卡片**：

| 卡片 | 内容 |
|------|------|
| IAMS 平台 | MQTT 连接状态（在线/离线）、最后同步时间 |
| 生物识别设备 | 各设备（虹膜/掌纹）的在线状态、IP 地址、凭证数量 |

**下发记录表格**：

显示最近 10 条凭证下发/删除记录，包含时间、凭证 ID、方向（下发/删除）、设备类型、状态和消息。每 60 秒自动刷新。

| 清空下发记录 | 清空所有下发记录（二次确认） |


### 5.2 凭证管理

**【截图：凭证管理页面】**

显示数据库中所有生物识别凭证的列表。

**表格列**：人员、用户编码、凭证 ID、类型、创建时间、操作。

**筛选功能**：可按凭证类型筛选（全部/虹膜/掌纹/密码/胁迫码）。

**操作**：

| 操作 | 说明 |
|------|------|
| 查看详细 | 弹出详情模态框，显示完整凭证信息 |
| 删除 | 仅虹膜/掌纹凭证可删除，通过 MQTT 发送删除指令 |

**凭证详情弹窗**：

显示凭证的完整信息，包括凭证 ID、类型、姓名、用户编码、人员类型、状态、识别模式、认证类型列表、箱号列表、虹膜/掌纹数据状态、创建/更新时间等。

### 5.3 MQTT 指令

**【截图：MQTT 指令记录页面】**

显示 IAMS 平台通过 MQTT 发送的指令记录。

**表格列**：时间、操作（添加/更新/删除）、凭证类型、用户编码、凭证 ID、认证模式、认证类型、箱号、显示信息。

**筛选与操作**：

| 功能 | 说明 |
|------|------|
| 日期筛选 | 按日期过滤记录 |
| 清空记录 | 清空所有 MQTT 指令记录 |

### 5.4 通行记录

**【截图：通行记录页面】**

显示认证成功的通行记录，每次认证成功后自动记录。

**表格列**：ID、用户编码、凭证 ID、认证类型、设备 ID、IAMS 响应状态、时间。

**IAMS 响应状态**：

| 状态 | 含义 |
|------|------|
| 成功（绿色） | IAMS 已确认收到通行记录 |
| 失败（红色） | IAMS 返回错误 |
| 未响应（灰色） | 尚未收到 IAMS 响应 |

**筛选功能**：支持按日期筛选和清除筛选。

### 5.5 系统设置

**【截图：系统设置页面】**

此页面包含所有系统配置项，分为以下几个区域：

#### 客户端配置（深色卡片）

显示当前后端服务器地址，点击"修改配置"可更改服务器地址并自动重启客户端。

| 配置项 | 说明 |
|--------|------|
| 服务器地址 | 一体化部署保持 `localhost:3001`，分离部署改为实际 IP |

#### 认证参数设置

| 配置项 | 默认值 | 范围 | 说明 |
|--------|--------|------|------|
| 认证超时时间 | 60 秒 | 5-300 秒 | 虹膜/掌纹验证的总超时时间 |
| 认证成功返回时间 | 10 秒 | 1-60 秒 | 认证成功后显示结果页的时长 |

#### 设备地址设置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 虹膜设备地址 | `http://192.168.3.202:9003` | 虹膜设备 HTTP 接口 |
| 掌纹设备地址 | `http://127.0.0.1:8080` | 掌纹设备 HTTP 接口 |

#### IAMS 上报设置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 认证终端设备 ID | `nuoyadev1` | 向 IAMS 平台上报通行记录的设备标识 |
| 通行记录最大保存条数 | 1000 条 | 超过此数量自动删除最旧记录 |

#### MQTT 连接设置

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| MQTT Broker 地址 | `mqtt://58.33.106.19:3881` | IAMS 平台的 MQTT 服务器地址 |
| MQTT 用户名 | `yq-device` | MQTT 连接用户名 |
| MQTT 密码 | `yqyq123!@#` | MQTT 连接密码 |

> **注意**：MQTT 配置保存后需要重启应用才能生效。其他设置保存后立即生效。

#### 设备工具

| 按钮 | 功能 |
|------|------|
| 重启后台服务 | 手动重启 Next.js 后台服务（当服务异常时可点击） |

## 7. 常见问题

### 7.1 身份编码输入后提示"库中无此用户编码信息"

该编码尚未在数据库中录入。请联系管理员在 IAMS 平台录入该用户的凭证。

### 7.2 虹膜/掌纹认证超时

- 确认生物识别设备已通电并正常工作
- 在"系统设置"中检查设备地址是否正确
- 虹膜认证：保持眼睛与摄像头平视，不要眨眼
- 掌纹认证：手掌平放，手指自然张开
- 可适当增加认证超时时间（系统设置 → 认证参数设置）

### 7.3 选择认证方式时无可用选项

该用户尚未录入任何生物识别凭证。请通过 IAMS 平台下发凭证。

### 7.4 设备管理显示设备离线

- 检查设备网络连接
- 在系统设置中确认设备 IP 地址和端口是否正确
- 虹默设备：`http://192.168.3.202:9003`
- 掌纹设备：`http://127.0.0.1:8080`

### 7.5 IAMS 平台离线

- 检查 MQTT 连接配置（Broker 地址、用户名、密码）
- 修改 MQTT 配置后需要重启应用
- 检查网络是否可访问 MQTT 服务器

### 7.6 修改设置后未生效

- 大部分设置保存后立即生效
- **MQTT 相关配置**（Broker、用户名、密码）需要重启应用
- **设备 ID 修改**：保存后会自动刷新 MQTT 订阅

### 7.7 数据备份

用户数据存储在 `%APPDATA%\noah-ark-electron\` 目录。如需备份：

1. 关闭应用
2. 复制 `%APPDATA%\noah-ark-electron\` 目录到备份位置
3. 恢复时将该目录复制回原位

### 7.8 覆盖安装

覆盖安装新版本不会丢失数据，因为数据存储在 AppData 目录而非安装目录。

### 7.9 后台服务断开

系统每 30 秒自动检测后端服务连接状态。如果服务异常断开（如任务管理器关闭、崩溃等），会弹出提示窗口：

- 点击 **"启动服务"**：自动重启后台服务，成功后页面刷新
- 点击 **"稍后"**：关闭提示，下次检测失败时再次弹出

也可在"系统设置"页面手动点击"重启后台服务"按钮。
