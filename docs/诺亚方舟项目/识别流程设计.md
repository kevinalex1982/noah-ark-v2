## 一、整体流程

```
待机页 → 点击进入 → 输入用户编码 → 凭证选择页 → 验证 → 认证结果
```

### 1.1 页面流程图

```
┌─────────────┐     点击      ┌─────────────────┐     输入完成     ┌─────────────────┐
│   待机页    │ ──────────→  │  输入用户编码页  │ ─────────────→  │   凭证选择页    │
│  (倒计时)   │              │   (倒计时返回)   │                 │   (倒计时返回)   │
└─────────────┘              └─────────────────┘                 └─────────────────┘
                                      │                                   │
                                      │                                   │ 选择凭证
                                      │                                   ↓
                                      │                          ┌─────────────────┐
                                      │                          │    验证页面     │
                                      │                          │  (密码/虹膜/掌纹) │
                                      │                          │   (倒计时返回)   │
                                      │                          └─────────────────┘
                                      │                                   │
                                      │                                   │ 验证结果
                                      ↓                                   ↓
                              ┌─────────────────────────────────────────────────┐
                              │                    认证结果页                    │
                              │                    (倒计时返回待机页)            │
                              └─────────────────────────────────────────────────┘
```

## 三、输入用户编码页面

### 3.1 功能
- 用户输入18位用户编码（personId）
- 验证格式（18位）
- 查询数据库获取该用户的凭证列表

### 3.2 查询逻辑

```typescript
// 1. 根据用户编码查询凭证列表
const credentials = await getCredentialsByPersonId(personId);

// 2. 如果没有凭证，提示"未注册"
if (credentials.length === 0) {
  return { error: '未注册用户' };
}

// 3. 获取第一个凭证的认证配置（所有人的配置应该一致）
const firstCredential = credentials[0];
const authModel = firstCredential.auth_model;        // 1=单凭证，2=多凭证组合
const authTypeList = firstCredential.auth_type_list; // "5,7,9"
const boxList = firstCredential.box_list;            // "rc10|rc20"

// 4. 过滤出有效凭证类型
const availableTypes = authTypeList.split(',').map(Number);
// 排除胁迫码(9)，胁迫码不显示在选择列表中
const displayTypes = availableTypes.filter(t => t !== 9);
```

### 3.3 倒计时
- 默认倒计时：60秒
- 倒计时结束返回待机页

## 四、凭证选择页

### 4.1 功能
- 显示该用户可用的验证方式
- 根据 authModel 决定是否显示组合认证选项
- 用户选择后进入对应的验证页面

### 4.2 凭证类型显示

| type | 名称 | 图标 | 说明 |
|------|------|------|------|
| 5 | 密码 | 🔢 | 密码输入 |
| 7 | 虹膜 | 👁️ | 虹膜识别 |
| 8 | 掌纹 | ✋ | 掌纹识别 |

**注意：胁迫码(9)不在选择列表中显示，只能在密码输入页面输入**

### 4.3 authModel 处理

#### 情况1：authModel = 1（单凭证识别）

**逻辑**：每个凭证单独验证，任一通过即可

**页面显示**：
```
┌─────────────────────────┐
│    请选择验证方式        │
├─────────────────────────┤
│  🔢 密码验证            │  ← 如果有密码凭证
│  👁️ 虹膜验证            │  ← 如果有虹膜凭证
│  ✋ 掌纹验证            │  ← 如果有掌纹凭证
└─────────────────────────┘
```

**验证规则**：
- 选择任一方式验证
- 验证通过 = 认证通过
- 验证失败 = 可选择其他方式或返回

#### 情况2：authModel = 2（多凭证组合识别）

**逻辑**：除单独验证外，还支持组合认证

**页面显示**：
```
┌─────────────────────────┐
│    请选择验证方式        │
├─────────────────────────┤
│  🔢 密码验证            │  ← 如果有密码凭证
│  👁️ 虹膜验证            │  ← 如果有虹膜凭证
│  ✋ 掌纹验证            │  ← 如果有掌纹凭证
├─────────────────────────┤
│  🔐 组合认证            │  ← 组合认证选项
│     (密码+虹膜)         │
└─────────────────────────┘
```

**组合认证规则**：
- 根据 authTypeList 确定组合顺序（排除胁迫码）
- 例：authTypeList = "5,7,9" → 先密码后虹膜
- 所有凭证都通过才算认证通过
- 中间任一失败 = 认证失败

**特殊说明**：
- 即使标记为组合认证，也可以选择单独验证
- 现场有工作人员协助，可能需要灵活处理

### 4.4 组合认证流程

```typescript
// 组合认证流程
async function combinedAuth(authTypeList: number[], credentials: Credential[]) {
  // 按authTypeList顺序，排除胁迫码
  const types = authTypeList.filter(t => t !== 9);
  const results = [];
  
  for (const type of types) {
    const credential = credentials.find(c => c.type === type);
    if (!credential) continue;
    
    // 显示对应的验证页面
    const result = await verifyCredential(type, credential);
    
    if (!result.success) {
      // 任一失败，整体失败
      return { success: false, failedAt: type };
    }
    
    results.push({ type, success: true });
  }
  
  // 全部通过
  return { success: true, results };
}
```

### 5.1 密码验证

**页面元素**：
- 密码输入框
- 数字键盘
- 倒计时
- 返回按钮

**特殊功能**：
- 支持输入胁迫码（如54321）
- 胁迫码触发特殊处理（静默报警）

**验证逻辑**：
```typescript
async function verifyPassword(inputPassword: string, credentials: Credential[]) {
  // 1. 检查是否为胁迫码
  const duressCredential = credentials.find(c => c.type === 9);
  if (duressCredential && inputPassword === duressCredential.content) {
    // 触发胁迫处理（静默）
    await triggerDuressAlert();
    return { success: true, isDuress: true };
  }
  
  // 2. 正常密码验证
  const pwdCredential = credentials.find(c => c.type === 5);
  if (pwdCredential && inputPassword === pwdCredential.content) {
    return { success: true };
  }
  
  return { success: false, error: '密码错误' };
}
```

## 六、虹膜识别流程

### 6.1 协议接口

**⚠️ 重要原则**：前端只调用自己的后端API，不直接调用设备API

```
前端 → 后端代理API(/api/device/iris/records) → 虹膜设备
```

**原因**：
- 安全问题：前端直接调用外部API很危险
- CORS问题：跨域请求不好处理
- Electron环境：需要服务器端日志

**后端代理API**：`POST /api/device/iris/records`

**请求参数**：
```json
{
  "startTime": 1594569600100,
  "endTime": 1594742399100,
  "count": 10
}
```

**响应字段**：
| 字段 | 说明 |
|------|------|
| `staffNum` | 工号（对应person_id） |
| `success` | 是否成功 true/false |
| `type` | 识别模式（1=虹膜） |
| `irisLeftScore` | 左眼比对分数 |
| `irisRightScore` | 右眼比对分数 |
| `createTime` | 识别时间 |

### 6.2 轮询机制设计

#### 核心策略：时间窗口滑动

**问题分析**：
- 虹膜设备主动识别，系统被动查询
- 需要区分"新产生的识别记录"和"历史记录"
- 网络传输有延迟，不能简单用"3秒内"判断

**解决方案**：滑动时间窗口

```
用户进入虹膜验证页面
        ↓
等待1秒后再开始查询（给设备准备时间）
        ↓
记录 startTime = 当前时间 - 1000（前一秒）
        ↓
首次查询：startTime = 验证开始前一秒
        ↓
如果查询到记录 → 检查是否匹配当前用户
        ↓
等待3秒
        ↓
下次查询：startTime = 上次查询发送时间
        ↓
循环直到超时或匹配成功
```

#### 时间参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `startTime` | 验证开始前一秒 | 首次查询用进入页面时间-1000，后续用上次请求发送时间 |
| `endTime` | 当前时间 | 每次查询时更新为当前时间 |
| `count` | 10 | 每次查询最多返回10条 |
| `needImages` | 0 | 不需要图片 |

#### 查询间隔

- **建议间隔**：3秒
- **原因**：
  1. 给设备足够时间处理识别
  2. 减少网络请求压力
  3. 3秒内产生的记录 + 网络延迟，足够覆盖

#### HTTP 超时设置

- **HTTP请求超时**：2秒
- **原因**：超过2秒的请求没有意义，直接放弃等待下一个查询周期
- **处理方式**：超时后立即返回，继续等待下一次轮询

#### 总超时设置

- **总超时**：从系统设置读取，默认60秒
- **超时后**：返回凭证选择页，提示"验证超时"

### 6.3 匹配逻辑（已实现）

```typescript
// 通过后端代理查询虹膜设备
async function verifyIris(personId: string): Promise<VerifyResult> {
  const startTime = Date.now();
  const timeout = countdown * 1000;
  let queryStartTime = startTime - 1000;

  // 等待1秒后再开始查询
  await sleep(1000);

  while (Date.now() - startTime < timeout) {
    try {
      // 通过后端代理查询虹膜设备
      const response = await fetch('/api/device/iris/records', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          startTime: queryStartTime,
          endTime: Date.now(),
          count: 10,
        }),
      });

      const result = await response.json();

      if (result.success && result.data) {
        const data = result.data;
        if (data.errorCode === 0 && data.body && data.body.length > 0) {
          for (const record of data.body) {
            // 匹配条件：staffNum = personId, success = true, type = 1（虹膜）
            if (record.staffNum === personId && record.success && record.type === 1) {
              return { success: true };
            }
          }
        }
      }
    } catch (error) {
      console.log('[虹膜] 查询失败:', error.message);
    }

    // 更新下次查询的开始时间
    queryStartTime = Date.now();

    // 等待轮询间隔
    await sleep(3000);
  }

  return { success: false, error: '验证超时' };
}
```

## 七、掌纹识别流程

### 7.1 协议接口

**⚠️ 重要原则**：前端只调用自己的后端API，不直接调用设备API

```
前端 → 后端代理API(/api/device/palm/query) → 掌纹设备
```

**原因**：
- 安全问题：前端直接调用外部API很危险
- CORS问题：跨域请求不好处理
- Electron环境：需要服务器端日志

**后端代理API**：`POST /api/device/palm/query`

**请求参数**：
```json
{
  "request": "103"
}
```

**request值说明**：
| 值 | 说明 |
|----|------|
| `103` | 开始识别 |
| `102` | 停止识别 |

**响应参数**：
| 字段 | 说明 |
|------|------|
| `code` | 100=未识别状态, 200=识别成功, 404=识别失败 |
| `des` | 识别结果描述，成功时为用户ID |

### 7.2 轮询机制设计（已实现）

#### 核心策略：周期查询 + 指令控制

**实际实现流程**：

```
用户进入掌纹验证页面
        ↓
发送103开始识别指令
        ↓
轮询查询识别状态（2秒间隔）
        ↓
收到响应判断code:
  - code=100 → 继续轮询（未识别）
  - code=200 → 暂停轮询 → 发送102停止 → 验证userId
  - code=404 → 继续轮询（识别失败）
        ↓
验证userId是否匹配当前用户:
  - 匹配成功 → 跳转成功页
  - 不匹配 → 发送103重新开始 → 继续轮询
        ↓
超时或用户退出 → 发送102停止指令
```

#### 停止指令(102)发送时机

| 场景 | 是否发送102 |
|------|------------|
| 识别成功验证用户前 | ✅ 发送 |
| 用户不匹配重新开始前 | ✅ 发送103重新开始 |
| 超时结束时 | ✅ 发送 |
| 用户点击重试 | ✅ 发送 |
| 用户点击返回 | ✅ 发送 |

#### 查询间隔

- **实际间隔**：2秒
- **原因**：
  1. 文档建议请求间隔大于200ms
  2. 2秒足够覆盖识别处理时间
  3. 减少不必要的网络请求

#### 总超时设置

- **总超时**：从系统设置读取，默认60秒
- **超时后**：发送102停止指令，返回凭证选择页

### 7.3 匹配逻辑（已实现）

**⚠️ 重要**：掌纹识别使用 `custom_id` 字段匹配

- 数据库中存储 `custom_id`（掌纹设备的userId，从特征数据提取）
- 识别时用 `des` 字段（设备返回的userId）与 `custom_id` 比对
- 匹配成功后从凭证记录中获取 `person_id`（用户编码）

**后端验证API**：`GET /api/auth/verify-palm?userId=xxx&identityId=xxx`

```typescript
// 前端调用
const verifyResponse = await fetch(`/api/auth/verify-palm?userId=${userId}&identityId=${identityId}`);
const verifyData = await verifyResponse.json();

if (verifyData.success && verifyData.match) {
  // 匹配成功，跳转成功页
} else {
  // 不匹配，重新发送103开始识别
}
```

## 八、认证结果处理

### 8.1 认证成功

- 显示欢迎信息
- 显示用户姓名
- 显示箱号（如果有）
- 倒计时后返回待机页

### 8.2 认证失败

- 显示失败原因
- 提供重试选项
- 倒计时后返回凭证选择页

### 8.3 胁迫码处理

**重要**：胁迫码验证"成功"但不触发报警显示
- 表面：显示认证成功
- 后台：静默发送警报
- 不让胁迫者察觉

## 十四、系统设置

### 14.1 设置页面

**路径**：`/dashboard/settings`

### 14.2 设置项

| 设置项 | 默认值 | 范围 | 说明 |

| 掌纹设备地址 | http://127.0.0.1:8080 | URL | 掌纹设备的endpoint |

### 14.3 设置存储结构

```json
{
  "authTimeout": 60,
  "successReturnTime": 10,
  "irisEndpoint": "http://192.168.3.202:9003",
  "palmEndpoint": "http://127.0.0.1:8080"
}
```

### 14.4 设置API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/settings` | 获取当前设置（仪表板使用） |
| GET | `/api/auth/settings` | 获取认证设置（Kiosk页面使用） |
| POST | `/api/settings` | 更新设置 |

### 14.5 实时生效机制

设置模块（`lib/settings.ts`）使用内存缓存 + 文件存储：

1. 读取时优先返回缓存
2. 更新时同时更新缓存和文件
3. 其他模块调用 `getAuthTimeoutMs()` 等函数获取实时设置
4. 设备地址通过 `getIrisEndpoint()` 和 `getPalmEndpoint()` 获取
